### Submission — Cursor Knight
### 👤 Name
Indrajeet Yadav

### 🧠 Theme
POINTER × RUNTIME

### 🔗 Itch.io Link
[Play the Game](https://indrajeety993648.itch.io/cursor-knight)

### 🎥 Gameplay Video
[Watch on Loom](https://www.loom.com/share/8031f1482a76477aad35db0b202a9fd8)

### 📁 Source Folder
[Submissions/CursorKnight](Submissions/CursorKnight/)

### 🛠️ Tech Stack
Unity 6000.0.72f1 (C#) · Built-in RP + Post Processing v2 · WebGL

### 📝 One-liner
A 2D side-scrolling runner where your cursor IS the samurai — hold **Z** and
drag mouse X to rewind time. 22 scripts, ~2700 LOC, no scene-authored
content (everything spawns at runtime via `[RuntimeInitializeOnLoadMethod]`).
Procedural audio via Python synthesis; procedural particles, shake, hitstop,
flash FX. 3 levels (Forest → Water → Ice Cave boss finale).

### 🎮 Controls
| Input | Action |
|---|---|
| Mouse Y | Steer knight vertically |
| Left-click | Jump (double for air jump) |
| Right-click | Slide |
| Hold **Z** + mouse X | Rewind ±2s / fast-forward +0.8s |

### 👤 Name  
Atharv Sanjay Jain  

### 🧠 Theme  
STRATEGY × CHAIN REACTIONS  

### 🔗 Itch.io Link  
[Play the Game](https://d-v-y-n-e.itch.io/chain-reaction-20)

### 🎥 Gameplay Video  
[Watch Gameplay](https://drive.google.com/file/d/19m3C7pyFxjS7aokIAIWBr1Hb0y8y3qwX/view?usp=sharing)

### 📁 Source Code  
[GitHub Repository](https://github.com/Atharv-2004/Chain-Reaction-Pro)

### 🛠️ Tech Stack  
Unity (C#) · WebGL · Unity UI System  

### 📝 One-liner  
A strategic 2-player Chain Reaction game with power-ups, an in-game economy system, and a best-of-3 match format — fully playable using mouse controls.

### 🎮 Controls  
| Input | Action |
|------|--------|
| Left Click | Place orb / Select cell |
| UI Buttons | Activate power-ups |
| Left Click (Laser mode) | Clear row |
| Right Click (Laser mode) | Clear column |

### ✨ Features  
- 2-player turn-based gameplay  
- Credit-based economy system  
- Power-ups: Laser, Trap, Block, Bomb, Swap, Diagonal  
- Chain reaction explosion mechanics  
- Best-of-3 (BO3) match system  
- Timer-based win condition  
- Multiple map layouts  

### 🧠 Highlights  
- Fully mouse-driven gameplay  
- Dynamic grid system  
- Real-time UI updates  
- Modular and scalable game architecture
